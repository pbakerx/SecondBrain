## The dominant framework: PARA + CODE (Tiago Forte)
- **PARA** organizes by _actionability_, not topic: **P**rojects (deadline) → **A**reas (ongoing) → **R**esources (reference) → **A**rchive (done). ← *This is the structure we used.*
- **CODE** is the workflow: **C**apture → **O**rganize → **D**istill → **E**xpress.
- Golden rule people repeat: **capture everything into one inbox, then process regularly** — move each item to a PARA bucket or delete it. (Your [Daily Log](../daily-log.md) is that inbox.)

## Tool camps (what people on X argue about)
- **Obsidian** — plain markdown, you own the files forever, graph view shows connections. Favored for long-term knowledge + AI-on-your-files workflows. *(Closest to what we built — markdown you own.)*
- **Notion** — database model fits PARA cleanly; nicer UI, syncs to mobile. Favored for starting out.
- **AI-native tools** (thesecondbrain.io, etc.) — auto-tag/summarize dropped-in content (videos, PDFs, threads). Good for creators drowning in inputs.

## The 2025/26 shift everyone's posting about
- Pairing a markdown vault with an **AI agent that reads and writes your files** — you view notes on one side, AI updates them on the other in real time. *(That's exactly this setup: your files + me.)*
- Creators use it to turn captured inputs (Looms, TikToks, threads) into reusable content.

## What we're doing differently (and why it fits you)
- Markdown-in-your-folder = zero lock-in, works with your existing AechTech repo, and I act as the always-on agent that files, distills, and nudges.
- Daily 7 AM review = the "process the inbox" habit, automated.
